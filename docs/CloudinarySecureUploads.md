# Cloudinary Secure Uploads Guide

This guide explains how to use both public and secure (signed URL) uploads with Cloudinary in our application.

## Overview

Our application supports two types of file uploads to Cloudinary:

1. **Public uploads**: Files that can be accessed directly via their Cloudinary URL without authentication
2. **Secure uploads**: Files that require a signed URL generated by our server to access

## Available Upload Middlewares

### Public Uploads (No Authentication Required)

```javascript
const { upload, imageUpload, documentUpload } = require("../middlewares/cloudinaryUpload");

// General purpose upload (auto-detects file type)
router.post("/upload", upload.single("file"), controller.handleUpload);

// Image-specific upload
router.post("/upload-image", imageUpload.single("image"), controller.handleImageUpload);

// Document-specific upload
router.post("/upload-document", documentUpload.single("document"), controller.handleDocumentUpload);
```

### Secure Uploads (Signed URLs Required)

```javascript
const { secureUpload, secureImageUpload, secureDocumentUpload } = require("../middlewares/cloudinaryUpload");

// Secure general purpose upload
router.post("/secure-upload", secureUpload.single("file"), controller.handleSecureUpload);

// Secure image upload
router.post("/secure-image", secureImageUpload.single("image"), controller.handleSecureImageUpload);

// Secure document upload
router.post("/secure-document", secureDocumentUpload.single("document"), controller.handleSecureDocumentUpload);
```

## Accessing Uploaded Files

### Public Files

Public files can be accessed directly using the URL returned by Cloudinary:

```javascript
router.get("/files/:id", async (req, res) => {
  const file = await File.findById(req.params.id);
  
  // For public files, you can use the URL directly
  res.json({ url: file.cloudinaryUrl });
});
```

### Secure Files

Secure files require a signed URL to access:

```javascript
const { generateSignedUrl } = require("../middlewares/cloudinaryUpload");

router.get("/secure-files/:id", async (req, res) => {
  const file = await SecureFile.findById(req.params.id);
  
  // Generate a signed URL that expires in 1 hour (default)
  const signedUrl = generateSignedUrl(file.cloudinaryPublicId);
  
  // Or with custom expiration time (e.g., 15 minutes)
  const shortLivedUrl = generateSignedUrl(file.cloudinaryPublicId, {
    expiresAt: Math.floor(Date.now() / 1000) + (15 * 60)
  });
  
  res.json({ url: signedUrl });
});
```

## Helper Utility for Mixed Content

If your application has both public and secure files, you can use the `getResourceUrl` helper:

```javascript
const { getResourceUrl } = require("../utils/cloudinaryUrlHelper");

router.get("/files/:id", async (req, res) => {
  const file = await File.findById(req.params.id);
  
  // This will automatically generate a signed URL if needed
  // or return the original URL for public files
  const accessUrl = getResourceUrl(file.cloudinaryUrl);
  
  res.json({ url: accessUrl });
});
```

## Best Practices

1. **Store public IDs, not URLs**: In your database, store the Cloudinary public ID rather than the full URL
2. **Generate signed URLs on demand**: Don't store signed URLs as they expire
3. **Set appropriate expiration times**: Make signed URLs expire as soon as reasonably possible
4. **Use secure uploads for sensitive content**: Medical records, personal documents, etc.
5. **Use public uploads for general content**: Marketing images, public documents, etc.

## Security Considerations

1. The Cloudinary API secret is only used on the server side and never exposed to clients
2. Signed URLs have a limited lifetime, after which they expire
3. Even if a signed URL is intercepted, it can only be used until it expires
4. Consider using IP restrictions in Cloudinary for additional security
